# 🛏️ Colchoeiro ERP — Sistema de Gestão para Colchoarias Sistema de gestão completo em HTML puro + Firebase + GitHub Pages. **100% gratuito** usando o plano Spark do Firebase (sem Firebase Storage). --- ## 💰 Por que é gratuito? | Serviço | Plano Spark (Gratuito) | Utilizado | |---------|------------------------|-----------| | Firebase Authentication | Ilimitado | ✅ Sim | | Cloud Firestore | 1 GB de dados, 50k leituras/dia | ✅ Sim | | Firebase Storage | **Cobrado** | ❌ Removido | | GitHub Pages | Ilimitado | ✅ Sim | **O que mudou em relação à versão anterior:** - Upload de foto de produto → substituído por campo de URL externa - Upload de logo da empresa → substituído por campo de URL externa - Upload de comprovante de pagamento → removido; apenas metadados são salvos (operadora, NSU, valor, parcelas) --- ## 📁 Estrutura de arquivos ``` colchoeiro/ ├── index.html ← Tela de login ├── dashboard.html ← App principal (carrega todos os módulos) ├── firestore.rules ← Regras de segurança do banco de dados ├── css/ │ └── main.css ← Design system completo ├── js/ │ ├── firebase-config.js ← Configuração do Firebase (sem Storage) │ ├── auth.js ← Autenticação e controle de sessão │ └── ui.js ← Utilitários de interface └── modules/ ├── dashboard.js ← Dashboard admin + dashboard vendedor ├── scanner.js ← Leitor de código de barras + comprovantes ├── products.js ← Cadastro e gestão de produtos ├── brands.js ← Marcas e margens ├── customers.js ← CRM de clientes ├── quotes.js ← Orçamentos com envio por WhatsApp ├── sales.js ← Vendas + financeiro + comissões + configurações └── ai-assistant.js ← Assistente IA (Claude) ``` --- ## 🚀 Como publicar no GitHub Pages (passo a passo) ### 1. Criar repositório no GitHub 1. Acesse **github.com** e faça login 2. Clique em **"+ New repository"** 3. Nome: `colchoeiro-erp` 4. Marque **Private** (privado) 5. Clique em **"Create repository"** ### 2. Fazer upload dos arquivos 1. Na página do repositório, clique em **"uploading an existing file"** 2. Arraste **todos os arquivos e pastas** do projeto 3. Clique em **"Commit changes"** ### 3. Ativar GitHub Pages 1. No repositório, clique em **Settings** (engrenagem) 2. No menu lateral, clique em **Pages** 3. Em **Source**, selecione **Deploy from a branch** 4. Branch: **main** | Folder: **/ (root)** 5. Clique em **Save** 6. Aguarde ~2 minutos → seu sistema estará em: `https://SEU-USUARIO.github.io/colchoeiro-erp/` --- ## 🔥 Configurar o Firebase ### 1. Aplicar as regras de segurança do Firestore 1. Acesse **console.firebase.google.com** 2. Selecione seu projeto Firebase 3. Clique em **Firestore Database → Regras** 4. Apague tudo e cole o conteúdo do arquivo `firestore.rules` 5. Clique em **Publicar** > ⚠️ **Não é necessário configurar Firebase Storage** — ele foi removido desta versão. ### 2. Habilitar autenticação por e-mail 1. No Firebase, clique em **Authentication → Sign-in method** 2. Clique em **E-mail/senha** 3. Ative o primeiro interruptor 4. Clique em **Salvar** ### 3. Criar o primeiro usuário administrador 1. No Firebase, clique em **Authentication → Usuários** 2. Clique em **Adicionar usuário** 3. Digite o e-mail e senha do admin 4. Copie o **UID** gerado (ex: `abc123...`) ### 4. Criar o documento do usuário no Firestore 1. No Firebase, clique em **Firestore Database** 2. Clique em **Iniciar coleção** → nome: `users` 3. ID do documento: cole o **UID** copiado acima 4. Adicione os campos: - `name` (string): `Seu Nome Completo` - `email` (string): `seu@email.com` - `role` (string): `admin` - `companyId` (string): `empresa-001` - `active` (boolean): `true` ### 5. Criar o documento da empresa 1. No Firestore, clique em **Iniciar coleção** → nome: `companies` 2. ID do documento: `empresa-001` (mesmo companyId acima) 3. Clique em **Salvar** 4. Dentro de `companies/empresa-001`, crie a subcoleção `settings` 5. Documento com ID `general`, campo `fantasyName` (string): nome da empresa ### 6. Atualizar as chaves do Firebase no projeto 1. Abra o arquivo `js/firebase-config.js` 2. Substitua os valores do objeto `firebaseConfig` pelas chaves do **seu** projeto Firebase 3. As chaves estão em: Firebase Console → Configurações do projeto → Seus apps → SDK --- ## 👥 Adicionar vendedores Para cada novo vendedor: 1. No Firebase, clique em **Authentication → Usuários → Adicionar usuário** 2. Digite o e-mail e senha do vendedor 3. Copie o **UID** gerado 4. No Firestore, crie o documento `/users/{uid}` com os campos: - `name` (string): Nome do vendedor - `email` (string): email@exemplo.com - `role` (string): `vendedor` (ou `gerente` / `financeiro`) - `companyId` (string): `empresa-001` - `active` (boolean): `true` > 💡 Alternativamente, o próprio admin pode criar usuários diretamente pelo sistema em **Configurações → Usuários → Convidar usuário**. --- ## 👥 Perfis de acesso | Perfil | Acesso | |--------|--------| | **admin** | Tudo, incluindo configurações, usuários e exclusão | | **gerente** | Dashboard geral, produtos, clientes, orçamentos, vendas, comissões | | **vendedor** | Seu dashboard, scanner, clientes, seus orçamentos | | **financeiro** | Dashboard, vendas, financeiro | --- ## 📱 Leitor de código de barras (mobile) O módulo Scanner usa a biblioteca **Quagga2** que é carregada automaticamente. - **Câmera traseira** do celular para leitura em tempo real - **Lanterna** integrada para ambientes escuros - **Leitores USB/Bluetooth** funcionam como teclado (plug and play) - **Modo Venda**: escaneia → monta carrinho → finaliza → registra comprovante - **Modo Cadastro**: escaneia → preenche formulário → salva no Firebase --- ## 🖼️ Como adicionar fotos de produtos (sem Storage) Como o Firebase Storage foi removido, as fotos são referenciadas por URL: 1. Hospede a imagem em um serviço gratuito: - **Google Drive**: faça upload, clique com botão direito → "Obter link" → altere para acesso público - **Imgur**: faça upload em imgur.com → copie o link direto da imagem - **ImgBB**: faça upload em imgbb.com → copie o link direto 2. No cadastro do produto, cole a URL no campo **"URL da foto do produto"** --- ## ⚠️ Sobre a segurança das chaves Firebase As chaves do Firebase (apiKey, projectId, etc.) são **públicas por design** — o Google as projeta para ficarem no front-end. A segurança real vem das **Security Rules** configuradas no passo acima. Para produção, ative também: - **App Check** no Firebase (bloqueia acessos fora do seu domínio) - **Domínios autorizados** em Authentication → Settings → Authorized domains --- ## 🤖 Assistente IA O módulo usa a API da Anthropic (Claude). Para ativar em produção: 1. Crie uma conta em **console.anthropic.com** 2. Gere uma API Key 3. Crie uma **Cloud Function** ou **proxy seguro** para não expor a chave no front-end 4. Atualize a URL do fetch no arquivo `modules/ai-assistant.js` --- ## 📊 Módulos implementados - ✅ Login / Logout / Recuperação de senha - ✅ Dashboard Admin (métricas, ranking, estoque crítico) - ✅ Dashboard Vendedor (suas vendas, produtos esquecidos, meta) - ✅ Scanner com câmera real (Quagga2) - ✅ Comprovantes de pagamento (metadados: operadora, NSU, valor, parcelas) - ✅ Produtos (CRUD completo com código de barras + URL de foto) - ✅ Marcas (margem e desconto máximo) - ✅ Clientes / CRM - ✅ Orçamentos (com envio por WhatsApp) - ✅ Vendas (com histórico e comprovantes) - ✅ Financeiro (receitas, comprovantes, formas de pagamento) - ✅ Comissões (percentual, meta, ranking) - ✅ Configurações (empresa, URL do logo, usuários, cores) - ✅ Assistente IA (Claude) - ✅ Controle de acesso por perfil (admin/gerente/vendedor/financeiro) - ✅ Responsivo (desktop, tablet e mobile) - ✅ **100% gratuito** no plano Spark do Firebase Colchoeiro ERP

// ============================================================ // COLCHOEIRO ERP — Regras de Segurança Firestore // Versão: Plano Gratuito (Spark) — Empresa Única // Firebase Storage: REMOVIDO // // Copie este conteúdo no console Firebase: // Firestore → Regras → Cole e publique // ============================================================ rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // ── Funções auxiliares ──────────────────────────────────── // Verifica se o usuário está autenticado function isAuth() { return request.auth != null; } // Pega o perfil do usuário no Firestore function userDoc() { return get(/databases/$(database)/documents/users/$(request.auth.uid)).data; } // Verifica se o usuário pertence à empresa function belongsToCompany(companyId) { return isAuth() && userDoc().companyId == companyId; } // Verifica o perfil do usuário function hasRole(companyId, role) { return belongsToCompany(companyId) && userDoc().role == role; } // Admin ou Gerente da empresa function isManager(companyId) { return belongsToCompany(companyId) && userDoc().role in ['admin', 'gerente']; } // Qualquer perfil com acesso comercial function isSalesOrAbove(companyId) { return belongsToCompany(companyId) && userDoc().role in ['admin', 'gerente', 'vendedor']; } // ── Empresas (companies) ────────────────────────────────── // Cada empresa é um "tenant" isolado match /companies/{companyId} { allow read: if belongsToCompany(companyId); allow write: if hasRole(companyId, 'admin'); // ── Usuários da empresa ───────────────────────────────── match /users/{userId} { // Usuário lê próprio perfil; admin lê todos allow read: if belongsToCompany(companyId) && (request.auth.uid == userId || isManager(companyId)); allow create: if hasRole(companyId, 'admin'); allow update: if hasRole(companyId, 'admin') || request.auth.uid == userId; allow delete: if hasRole(companyId, 'admin'); } // ── Marcas ────────────────────────────────────────────── match /brands/{brandId} { allow read: if belongsToCompany(companyId); allow write: if isManager(companyId); } // ── Produtos ──────────────────────────────────────────── match /products/{productId} { allow read: if belongsToCompany(companyId); allow create: if isManager(companyId); allow update: if isManager(companyId); allow delete: if hasRole(companyId, 'admin'); } // ── Clientes ──────────────────────────────────────────── match /customers/{customerId} { allow read: if isSalesOrAbove(companyId); allow write: if isSalesOrAbove(companyId); allow delete: if isManager(companyId); } // ── Orçamentos ────────────────────────────────────────── match /quotes/{quoteId} { // Vendedor só lê/edita seus próprios orçamentos allow read: if belongsToCompany(companyId) && (isManager(companyId) || resource.data.sellerId == request.auth.uid); allow create: if isSalesOrAbove(companyId) && request.resource.data.sellerId == request.auth.uid; allow update: if isSalesOrAbove(companyId) && (isManager(companyId) || resource.data.sellerId == request.auth.uid); allow delete: if isManager(companyId); } // ── Itens do orçamento ────────────────────────────────── match /quoteItems/{itemId} { allow read: if isSalesOrAbove(companyId); allow write: if isSalesOrAbove(companyId); } // ── Vendas ────────────────────────────────────────────── match /sales/{saleId} { allow read: if belongsToCompany(companyId) && (isManager(companyId) || hasRole(companyId, 'financeiro') || resource.data.sellerId == request.auth.uid); allow create: if isSalesOrAbove(companyId); allow update: if isManager(companyId) || hasRole(companyId, 'financeiro'); allow delete: if hasRole(companyId, 'admin'); } // ── Comprovantes de pagamento ─────────────────────────── match /paymentProofs/{proofId} { allow read: if belongsToCompany(companyId) && (isManager(companyId) || hasRole(companyId, 'financeiro') || resource.data.uploadedBy == request.auth.uid); allow create: if isSalesOrAbove(companyId); allow update: if isManager(companyId) || hasRole(companyId, 'financeiro'); allow delete: if isManager(companyId); } // ── Estoque ───────────────────────────────────────────── match /inventory/{itemId} { allow read: if belongsToCompany(companyId); allow write: if isManager(companyId); } // ── Movimentações de estoque ──────────────────────────── match /inventoryMovements/{movId} { allow read: if belongsToCompany(companyId); allow create: if isSalesOrAbove(companyId); allow update, delete: if isManager(companyId); } // ── Financeiro ────────────────────────────────────────── match /financial/{docId} { allow read: if isManager(companyId) || hasRole(companyId, 'financeiro'); allow write: if isManager(companyId) || hasRole(companyId, 'financeiro'); } // ── Comissões ─────────────────────────────────────────── match /commissions/{comId} { allow read: if belongsToCompany(companyId) && (isManager(companyId) || resource.data.sellerId == request.auth.uid); allow write: if isManager(companyId); } // ── Configurações da empresa ──────────────────────────── match /settings/{settingId} { allow read: if belongsToCompany(companyId); allow write: if hasRole(companyId, 'admin'); } // ── Logs de auditoria (somente leitura para admin) ────── match /auditLogs/{logId} { allow read: if hasRole(companyId, 'admin'); allow create: if belongsToCompany(companyId); allow update, delete: if false; // logs são imutáveis } } // ── Coleção global de usuários ────────────────────────────── // Usuário lê o próprio perfil; admin pode criar novos usuários match /users/{userId} { allow read: if isAuth() && (request.auth.uid == userId || userDoc().role == 'admin'); allow create: if isAuth() && userDoc().role == 'admin'; allow update: if isAuth() && (request.auth.uid == userId || userDoc().role == 'admin'); allow delete: if false; } // Bloqueia tudo que não foi explicitamente permitido match /{document=**} { allow read, write: if false; } } }
